# Система распознавания параметров оптических вихрей (OV-PRS)

**[English](README.md) | [中文](README_zh.md) | [日本語](README_ja.md)**

Программное средство Optical Vortex Parameter Recognition System (OV-PRS) разработано для решения специализированной задачи распознавания параметров оптических вихрей семейства Лагерра–Гаусса по изображениям интенсивности. Программа использует методы машинного обучения для анализа входных данных, полученных с камеры экспериментальной установки, и автоматически определяет параметры LG‑моды, включая радиальный и азимутальный индексы. На основе распознанных параметров может быть вычислен и топологический заряд оптического вихря.

Реализация ориентирована на работу в условиях турбулентных искажений, неизбежно возникающих в реальных оптических системах. Используемая нейросетевая модель обучена на данных, моделирующих различные уровни турбулентности, что обеспечивает устойчивое распознавание параметров даже при значительных искажениях интенсивностного профиля.

Программное средство создано под конкретную исследовательскую задачу, однако архитектура решения позволяет адаптировать его под другие типы оптических мод, иные параметры пучков или отличающиеся условия эксперимента. При необходимости система может быть модифицирована под требования пользователя: изменён набор распознаваемых параметров, расширена модель, интегрирована новая камера или формат данных, а также обучена под специфические условия распространения света.

## Функциональность

- **Распознавание параметров**: Определение радиального индекса (n), азимутального индекса (m) и топологического заряда (TC)
- **Режимы работы**:
  - Захват изображений с веб-камеры в реальном времени
  - Загрузка отдельных изображений с диска
  - Пакетная обработка множества изображений
- **Экспорт результатов**: Сохранение результатов распознавания в CSV формате
- **Веб-интерфейс**: Работа через браузер на localhost
- **Мультиязычность**: Английский, русский, китайский, японский

## Требования к системе

- Python 3.8+
- PyTorch 2.0+
- OpenCV 4.8+
- Flask 2.3+
- NumPy, Pillow, torchvision

**Примечание**: В связи с узконаправленностью приложения для распознавания оптических вихрей, предъявляются специфические требования к нейросетевым моделям.

## Использование

### Запуск приложения
```bash
python main.py
```

После запуска откройте браузер и перейдите по адресу: http://127.0.0.1:5000

### Режимы работы

1. **Загрузка модели**:
   - Нажмите "Загрузить модель" и выберите файл предобученной модели (.pth или .pt)
   - Модель будет загружена и готова к работе

2. **Работа с камерой**:
   - Нажмите "Подключить камеру"
   - Приложение начнет захват изображений с веб-камеры в реальном времени
   - Распознавание параметров происходит автоматически
   - Результаты отображаются в правой панели
   - Нажмите "Остановить камеру" для завершения работы

3. **Загрузка изображений**:
   - Нажмите "Загрузить изображение" для выбора одного файла
   - Изображение будет автоматически обработано
   - Результаты распознавания появятся сразу после загрузки

4. **Пакетная обработка**:
   - Нажмите "Пакетная загрузка" для выбора нескольких изображений
   - Все изображения будут обработаны последовательно
   - Результаты добавятся в таблицу истории

5. **Просмотр и экспорт результатов**:
   - Нажмите "Показать таблицу" для просмотра всех результатов
   - Нажмите "Экспорт CSV" для сохранения результатов в файл
   - Нажмите "Очистить результаты" для удаления всех данных

## Нейросетевая модель

### Требования к модели
- Формат: PyTorch (.pth или .pt)
- Выход: кортеж из двух тензоров (предсказания для n и m)
- Вход: RGB изображение, 227x227 пикселей

### Диапазоны параметров
- **Радиальный индекс (n)**: 7 классов (от 1 до 7)
- **Азимутальный индекс (m)**: 7 классов (от 2 до 8)
- **Топологический заряд (TC)**: вычисляется как TC = m - n (всегда >= 1)

## Подготовка моделей

Приложение поддерживает загрузку предобученных моделей PyTorch в формате state_dict (словарь весов).

### Формат сохранения модели

Сохранение только весов модели (рекомендуется разработчиками PyTorch):

```python
import torch

# Сохранение state_dict
torch.save(model.state_dict(), 'model_state_dict.pth')

# Или с оберткой
torch.save({'model_state_dict': model.state_dict()}, 'model_state_dict.pth')
```

Приложение использует архитектуру AlexNetLG для загрузки state_dict. Убедитесь, что архитектура вашей модели соответствует архитектуре AlexNetLG приложения.

## Формат результатов

Результаты распознавания сохраняются в CSV формате со следующими столбцами:
- `filename`: Имя файла изображения
- `n`: Радиальный индекс
- `m`: Азимутальный индекс
- `TC`: Топологический заряд

## Устранение неисправностей

### Проблемы с камерой
- Убедитесь, что веб-камера подключена и работает
- Проверьте, что другие приложения не используют камеру
- Перезапустите приложение при возникновении ошибок

### Проблемы с загрузкой модели
- Убедитесь, что файл модели имеет формат .pth или .pt
- Проверьте, что модель совместима с версией PyTorch
- Проверьте консольный вывод на наличие ошибок

### Проблемы с распознаванием
- Убедитесь, что модель загружена корректно
- Проверьте качество изображений
- Убедитесь, что формат изображения поддерживается

### Ошибки запуска
- Установите все зависимости из requirements.txt
- Проверьте версию Python (требуется 3.8+)
- Убедитесь, что порт 5000 не занят другим приложением
