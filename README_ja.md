# 光渦パラメータ認識システム (OV-PRS)

**[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md)**

光渦パラメータ認識システム (OV-PRS) は、強度画像からラゲール・ガウス光渦のパラメータを認識する専門的なタスクを解決するために設計されています。このプログラムは機械学習手法を使用して、実験装置のカメラから取得した入力データを分析し、LGモードパラメータ（半径インデックスと方位角インデックスを含む）を自動的に決定します。認識されたパラメータに基づいて、光渦のトポロジカル電荷も計算できます。

この実装は、実際の光学システムで必然的に発生する乱流歪み条件下での動作を目指しています。使用されるニューラルネットワークモデルは、さまざまな乱流レベルをシミュレートするデータでトレーニングされており、強度プロファイルに顕著な歪みがあっても堅牢なパラメータ認識を保証します。

このソフトウェアは特定の研究タスクのために作成されましたが、ソリューションアーキテクチャにより、他のタイプの光学モード、異なるビームパラメータ、または異なる実験条件に適応できます。必要に応じて、システムはユーザーの要件に合わせて変更できます：認識パラメータセットの変更、モデルの拡張、新しいカメラまたはデータ形式の統合、および特定の光伝播条件でのトレーニングが可能です。

## 機能

- **パラメータ認識**: 半径インデックス (n)、方位角インデックス (m)、トポロジカル電荷 (TC) の決定
- **動作モード**:
  - Webカメラからのリアルタイム画像キャプチャ
  - ディスクからの単一画像のロード
  - 複数画像のバッチ処理
- **結果のエクスポート**: 認識結果をCSV形式で保存
- **Webインターフェース**: ブラウザを介したlocalhostでの動作
- **多言語サポート**: 英語、ロシア語、中国語、日本語

## システム要件

- Python 3.8+
- PyTorch 2.0+
- OpenCV 4.8+
- Flask 2.3+
- NumPy, Pillow, torchvision

**注**: アプリケーションは光渦認識に特化しているため、ニューラルネットワークモデルに特定の要件があります。

## 使用方法

### アプリケーションの起動
```bash
python main.py
```

起動後、ブラウザを開いて次のURLに移動してください: http://127.0.0.1:5000

### 動作モード

1. **モデルのロード**:
   - 「モデルをロード」をクリックし、事前トレーニング済みモデルファイル（.pthまたは.pt）を選択します
   - モデルがロードされ、使用準備が整います

2. **カメラ操作**:
   - 「カメラを接続」をクリックします
   - アプリケーションがWebカメラからリアルタイムで画像のキャプチャを開始します
   - パラメータ認識が自動的に行われます
   - 結果が右パネルに表示されます
   - 「カメラを停止」をクリックして終了します

3. **画像のロード**:
   - 「画像をロード」をクリックして単一のファイルを選択します
   - 画像が自動的に処理されます
   - 認識結果はロード後すぐに表示されます

4. **バッチ処理**:
   - 「バッチロード」をクリックして複数の画像を選択します
   - すべての画像が順次処理されます
   - 結果が履歴テーブルに追加されます

5. **結果の表示とエクスポート**:
   - 「テーブルを表示」をクリックしてすべての結果を表示します
   - 「CSVエクスポート」をクリックして結果をファイルに保存します
   - 「結果をクリア」をクリックしてすべてのデータを削除します

## ニューラルネットワークモデル

### モデル要件
- 形式: PyTorch (.pthまたは.pt)
- 出力: 2つのテンソルのタプル（nとmの予測）
- 入力: RGB画像、227x227ピクセル

### パラメータ範囲
- **半径インデックス (n)**: 7クラス（1から7）
- **方位角インデックス (m)**: 7クラス（2から8）
- **トポロジカル電荷 (TC)**: TC = m - nとして計算（常に>= 1）

## モデルの準備

アプリケーションは、state_dict形式（重み辞書）で事前トレーニング済みのPyTorchモデルのロードをサポートしています。

### モデル保存形式

モデルの重みのみを保存（PyTorch開発者推奨）:

```python
import torch

# state_dictの保存
torch.save(model.state_dict(), 'model_state_dict.pth')

# またはラッパーを使用
torch.save({'model_state_dict': model.state_dict()}, 'model_state_dict.pth')
```

アプリケーションはAlexNetLGアーキテクチャを使用してstate_dictをロードします。モデルアーキテクチャがアプリケーションのAlexNetLGアーキテクチャと一致していることを確認してください。

## 結果形式

認識結果はCSV形式で保存され、次の列が含まれます:
- `filename`: 画像ファイル名
- `n`: 半径インデックス
- `m`: 方位角インデックス
- `TC`: トポロジカル電荷

## トラブルシューティング

### カメラの問題
- Webカメラが接続され正常に動作していることを確認してください
- 他のアプリケーションがカメラを使用していないことを確認してください
- エラーが発生した場合はアプリケーションを再起動してください

### モデルロードの問題
- モデルファイルが.pthまたは.pt形式であることを確認してください
- モデルがPyTorchバージョンと互換性があることを確認してください
- コンソール出力にエラーがないか確認してください

### 認識の問題
- モデルが正しくロードされていることを確認してください
- 画像品質を確認してください
- 画像形式がサポートされていることを確認してください

### 起動エラー
- requirements.txtからすべての依存関係をインストールしてください
- Pythonバージョンを確認してください（3.8+が必要です）
- ポート5000が他のアプリケーションによって占有されていないことを確認してください

## 技術サポート

問題が発生した場合:
1. アプリケーションのコンソール出力を確認してください
2. すべての依存関係が正しくインストールされていることを確認してください
3. モデル形式が要件を満たしていることを確認してください
